# 📌 **Jogo de Forca Online – Levantamento de Requisitos**  

## 🎯 **Descrição Geral**  
O jogo de forca online permitirá que jogadores participem de partidas individuais ou multiplayer, adivinhando palavras dentro de um limite de tentativas. Ele será acessível via navegador e poderá ser jogado em tempo real.  

## ✅ **Requisitos Funcionais**  
1. O sistema deve permitir partidas no modo **single-player** e **multiplayer**.  
2. O sistema deve permitir que jogadores **entrem em salas** para partidas multiplayer.  
3. O sistema deve permitir que um jogador crie uma **sala privada** com senha.  
4. O sistema deve permitir que o criador da sala escolha o **tema das palavras** (exemplo: animais, filmes, tecnologia etc.).  
5. O sistema deve selecionar uma palavra aleatória baseada no tema escolhido.  
6. O sistema deve exibir a palavra oculta com traços representando letras não descobertas.  
7. O sistema deve permitir que o jogador escolha **uma letra por vez**.  
8. O sistema deve indicar letras corretas e incorretas.  
9. O sistema deve exibir um **contador de tentativas restantes**.  
10. O sistema deve indicar quando um jogador vence ou perde a partida.  
11. No modo multiplayer, o sistema deve alternar os turnos entre os jogadores.  
12. O sistema deve permitir a exibição de um **ranking** com pontuação dos jogadores.  
13. O sistema deve oferecer um **chat** para comunicação entre os jogadores durante as partidas.  
14. O sistema deve permitir o login via **Google ou e-mail/senha** para salvar o progresso.  
15. O sistema deve permitir a personalização do avatar do jogador.  

## 🚀 **Requisitos Não Funcionais**  
1. O sistema deve carregar a tela de jogo em **menos de 2 segundos**.  
2. O sistema deve ser desenvolvido com **arquitetura responsiva**, suportando telas de desktop e mobile.  
3. O servidor deve suportar pelo menos **100 jogadores simultâneos** sem perda de desempenho.  
4. O sistema deve garantir a segurança das credenciais dos jogadores usando **criptografia de senha**.  
5. O sistema deve ter um tempo de resposta de no máximo **200ms** para interações durante o jogo.  
6. O chat do jogo deve ser **moderado automaticamente** para evitar mensagens ofensivas.  
7. O sistema deve ser implementado em uma arquitetura escalável baseada em **microserviços** ou **serverless**.
